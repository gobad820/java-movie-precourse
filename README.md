## 기능 구현 전략

## AI를 이용해서 학습한 점

- JUnit5에서 도입된 @Nested 어노테이션을 통한 구조화된 테스트 구현
- JUnit5에서 @Parameterized Test를 이용해 코드 중복 제거
    - @ValueSource를 통해 동일 주제의 테스트의 여러 케이스 생성 가능
- 테스트 과정 중 .trim.empty()의 경우
